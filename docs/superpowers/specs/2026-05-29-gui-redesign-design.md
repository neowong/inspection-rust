# 网络设备巡检系统 v3.1 — 桌面 GUI 重构设计

日期: 2026-05-29
状态: 待审批

## 目标

将原 WEB 风格 Tauri 应用全面重构为桌面原生风格的工程师工具：
- 纯 Tauri v2 技术栈（Rust + React），保留 webview 渲染与浏览器打印 PDF 能力
- 前端交互完全按桌面 GUI 模式设计，不沿用 WEB 页面风格
- 报告输出从 HTML 拼接改为 Markdown 生成 → 前端渲染 → 浏览器打印 PDF
- 移除 AI 对话、离线巡检、定时任务、Web 设备巡检；命令库合并入模板编辑
- 其余功能与原版保持一致

## 3. 报告管道

```
巡检完成
  → AI 分析 (ai_inspection.rs) 生成 command_judgments + summary
  → Rust 收集: 设备信息 + 命令输出 + 评判结果
  → report_generator::build_markdown() → String
  → 保存为 .md 文件到 reports/ 目录
  → 前端: 读取 .md → React 渲染 (react-markdown / marked.js)
  → 预览: 在 WebView 中显示 HTML 渲染结果
  → 导出 PDF: window.print() 浏览器打印 API
```

Markdown 报告模板结构：

```markdown
# {{device_name}} 巡检报告

## 基本信息
| 项目 | 值 |
|------|----|
| 设备名称 | {{name}} |
| IP 地址 | {{ip}} |
...

## 巡检记录
| 序号 | 巡检项目 | 命令输出 | 评判结论 |
|------|---------|---------|---------|
| 1 | display version | ... | [OK] 运行正常 |
...

## 总结
{{summary}}
```

### 对比当前方案

| | 当前（HTML 字符串拼接） | 新方案（Markdown） |
|---|---|---|
| 生成代码 | 200+ 行 format! 嵌套 | 50 行模板式字符串 |
| 可读性 | HTML 源码不可读 | .md 文件可直接查看 |
| PDF 导出 | 无 | window.print() 一行 |
| 可 diff | 差 | 好 |
| 维护 | 改 CSS 要改 Rust 代码 | 样式和内容分离 |

## 4. 前端 GUI 风格设计

### 4.1 窗口布局（AppShell）

```
┌────────────────────────────────────────────────┐
│  文件  操作  视图  帮助        _  □  X 菜单栏     │
├────────┬───────────────────────────────────────┤
│ 仪表盘  │                                        │
│ 设备管理 │          内容区域                      │
│ 巡检模板 │     (页面标题 + 工具栏 + 数据区)        │
│ 执行巡检 │                                        │
│ 巡检报告 │                                        │
│ AI 配置  │                                        │
│ 系统设置 │                                        │
│        │                                        │
├────────┴───────────────────────────────────────┤
│ ✅ 就绪  │ 上次巡检: 2024-01-15 10:30  设备: 5  状态栏 │
└────────────────────────────────────────────────┘
```

**桌面交互要点**：
- 菜单栏：文件(新建/导入/导出/退出)、操作(开始巡检/刷新状态)、视图(切换页面/全屏)、帮助(关于)
- 侧边栏：图标+文字，支持折叠至仅图标
- 状态栏：左侧状态文本，右侧统计数据，支持显示后台任务进度
- 全局键盘快捷键：Ctrl+N 新建、Ctrl+S 保存、F5 刷新、Ctrl+F 搜索

### 4.2 各页面交互设计

#### 仪表盘
- 顶部：4 个状态卡片行（设备总数/在线/离线/最近批次），紧凑排列
- 左下：最近巡检批次列表（双击行跳转报告详情）
- 右下：最近设备状态变更日志时间线
- 卡片和列表项支持双击打开详情

#### 设备管理
- **分组树+表格式**：左侧设备分组树（network/system），右侧表格
- 表格：紧凑行高（28px），列头可拖拽排序、点击筛选
- **右键菜单**：
  ```
  ┌──────────────┐
  │ 编辑设备      │
  │ 手动巡检      │
  │ ──────────── │
  │ 复制设备      │
  │ 导出配置      │
  │ ──────────── │
  │ 删除设备      │
  └──────────────┘
  ```
- 双击行打开编辑对话框（模态窗口）
- 批量选择：Ctrl+点击多选 / Shift+点击范围选
- 工具栏：添加、刷新状态、批量删除、搜索（实时过滤，非按钮触发）
- 新增/编辑：模态对话框（非页面跳转），表单布局两列紧凑对齐

#### 巡检模板
- **三栏布局**：
  ```
  ┌──────────┐ ┌─────────────────┐ ┌──────────────┐
  │ 模板列表   │ │ 模板详情+命令编辑  │ │ 命令库面板    │
  │           │ │                 │ │ (可从左侧     │
  │ 模板1     │ │ 名称: [______]  │ │  拖拽命令      │
  │ 模板2  ◀  │ │ 厂商: [______]  │ │  到中间区域)   │
  │ 模板3     │ │                 │ │              │
  │           │ │ 命令列表:        │ │ 搜索: [___]  │
  │ [+新建]   │ │ ├ display ver   │ │ ├ 命令1  [+] │
  │           │ │ ├ display dev   │ │ ├ 命令2  [+] │
  │           │ │ └ ...           │ │ └ ...        │
  │           │ │ [保存] [另存为]  │ │              │
  └──────────┘ └─────────────────┘ └──────────────┘
  ```
- 左侧模板列表支持右键（重命名/删除/复制）
- 中间命令列表支持拖拽排序、右键移除
- 右侧命令库有搜索过滤 + 标签筛选（厂商/分类）
- 命令库管理操作直接在此面板完成（添加/编辑/删除命令）

#### 执行巡检
- **类似 CI/CD Pipeline 视图**：
- 顶部：选择设备（复选框树）、选择模板、开始/暂停/停止按钮
- 主区域：执行进度条 + 设备列表（树形展开）
  ```
  ┌────────────────────────────────────────────────┐
  │ 进度: ████████░░░░░░░░ 3/5 (60%)               │
  │                                                │
  │ │ 核心交换机-Core01             ✅ 完成 (12s)    │
  │ │ 汇聚交换机-Agg02             🔄 执行中...     │
  │ │ 数据库服务器-DB01            ❌ 连接失败       │
  │ │ 防火墙-FW01                   ⏳ 等待中        │
  │ │ 接入交换机-ACC03              ⏳ 等待中        │
  └────────────────────────────────────────────────┘
  ```
- 点击已完成设备可展开命令执行详情（内嵌的迷你终端输出）
- 巡检完成后弹出通知 + 自动跳转报告页（可配置）

#### 巡检报告
- **双视图切换**：报告列表视图 / 报告预览视图
- 报告列表：
  - 按批次分组显示（树形：批次 → 设备报告）
  - 列：设备名称、巡检时间、AI 整体判断（彩色标签）、操作
  - 支持多选批量导出 PDF、批量删除
- 报告预览（点击单条记录进入）：
  ```
  ┌────────────────────────────────────────────────┐
  │ ← 返回列表    [导出 PDF] [打印] [导出 Markdown]  │
  ├────────────────────────────────────────────────┤
  │                                                │
  │  # 核心交换机-Core01 巡检报告                   │
  │                                                │
  │  ## 基本信息                                    │
  │  | 项目 | 值 |                                  │
  │  |------|----|                                  │
  │  ...                                           │
  │                                                │
  │  ## 巡检记录                                    │
  │  | 序号 | 项目 | 输出 | 结论 |                   │
  │  |------|------|------|------|                  │
  │  ...                                           │
  └────────────────────────────────────────────────┘
  ```
- 预览区为渲染后的 Markdown（表格、彩色标签、代码块）
- AI 分析结果可折叠展开
- 导出 PDF 直接用浏览器 `window.print()` 打印 Markdown 渲染页面

#### AI 配置
- 表单式布局，两列对齐
- 模型提供商下拉 + API 密钥（密码框 + 显示/隐藏切换）+ 模型 ID + Base URL
- 底部：测试连接按钮（功能按钮，点击显示连接结果 toast）
- 配置列表在下半部分的表格中（可多行激活标记）

#### 系统设置
- 选项卡式分组（报告 / 网络 / 加密 / 关于）
- 报告：最大输出行数(numeric)、报告保存路径(带浏览按钮)
- 网络：SSH 并发数、连接超时
- 加密：Master Password 重设
- 关于：版本号、致谢、License

### 4.3 GUI 交互原则

| WEB 模式（砍掉） | GUI 模式（新） |
|------------------|----------------|
| 页面跳转 + 面包屑 | 单页面内的模态对话框/面板切换 |
| 表格顶部的搜索按钮 | 即时搜索输入框（实时过滤） |

- **报告列表右击**：查看 / 导出 PDF / 打印 / — / 删除
- **命令列表右击**：编辑命令 / 复制到模板 / — / 从模板移除

## 5. 后端优化

### 5.1 CRUD 样板消除

用声明式宏减少 10 个模块的重复代码：

```rust
// 宏定义（在 lib.rs 中）
macro_rules! crud_commands {
    ($module:ident, $type:ty, $list_sql:expr, $get_sql:expr) => {
        #[tauri::command]
        fn list_devices(state: tauri::State<AppState>) -> Result<Vec<$type>, String> {
            let db = state.db.lock();
            query_all::<$type>(&db, $list_sql)
        }
        // ... get, create, update, delete, batch_delete
    };
}
```

### 5.2 错误处理统一

```rust
#[derive(thiserror::Error, Debug)]
pub enum AppError {
    #[error("数据库错误: {0}")]    Db(#[from] rusqlite::Error),
    #[error("未找到: {0}")]        NotFound(String),
    #[error("SSH 错误: {0}")]      Ssh(String),
    #[error("加密错误: {0}")]      Crypto(String),
    #[error("序列化错误: {0}")]    Serde(#[from] serde_json::Error),
}
impl From<AppError> for String { ... } // Tauri 自动转换
```

### 5.3 SSH 执行异步化

`inspection_runner.run_batch()` 中每个设备 SSH 操作放入 `tokio::task::spawn_blocking`，通过网络事件 (`app.emit()`) 向前端推送实时进度。
