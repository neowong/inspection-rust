## 1. 安装与启动

### Windows

1. 从 [GitHub Releases](https://github.com/neowong/inspection-rust/releases) 下载最新版 `.exe` 安装包（NSIS 安装程序，自包含 WebView2 引导）
2. 双击运行安装程序，按向导完成安装
3. 首次启动前，**确保已安装 Microsoft Edge WebView2 Runtime**——Win11 自带，Win10 若缺失安装包会自动引导安装
4. 从桌面快捷方式或开始菜单启动 **AI巡检助手**

> **若启动无反应**：检查 `%TEMP%\inspection-debug.log`，或参见 [第 11 节](#11-常见问题排查)

### Linux

```bash
# 从 Releases 下载 .deb 包
sudo dpkg -i inspection-rust_3.40.16_amd64.deb
# 终端启动
inspection-rust
```

### 升级

直接用新版本安装包覆盖安装即可，**无需卸载、无需删除数据库**。程序启动时自动执行增量数据库迁移，旧版本录入的设备、模板、历史巡检数据完整保留。

## 3. 设备管理

管理所有被巡检的设备：网络设备、Linux 服务器、数据库。

### 3.1 设备类型与厂商

| 设备类型 | 可选厂商 |
|---------|---------|
| 交换机 / 路由器（网络设备）| H3C、华为、思科、锐捷、飞塔、其它 |
| 防火墙 / 负载均衡（安全设备）| 同上 |
| 服务器 | Linux、Ubuntu、CentOS、Rocky、Debian、其它 |
| 数据库 | MySQL、PostgreSQL、Oracle、SQL Server、达梦、Redis、MongoDB、其它 |

> **IP 唯一性按设备类型**：同一 IP 可添加不同类型的设备（如一台 Linux 服务器 + 其上的 MySQL 数据库都用 192.168.1.10），同类型内 IP 不可重复。

### 3.2 添加设备

点击右上角 **"+ 添加设备"**，填写：

**基础信息（所有设备）**

| 字段 | 说明 |
|------|------|
| 设备名称 | 自定义标识，全局唯一 |
| IP 地址 | 管理 IP |
| 设备类型 | 交换机/路由器/防火墙/负载均衡/服务器/数据库 |
| 厂商 | 见上表 |
| SSH 端口 | 默认 22 |
| SSH 用户名 / 密码 | 登录凭据（密码加密存储）|
| 关联模板 | 绑定巡检模板（必填）|

**服务器/数据库额外字段**：主机名、发行版本号、内核版本、CPU 核心数、内存（均自动检测，可手填）

**数据库专属字段**

| 字段 | 说明 |
|------|------|
| 部署方式 | 包安装 / Docker 容器 / Podman 容器 / Kubernetes |
| 容器名 | **容器部署时必填**，如 `mysql`；K8s 填 `deploy/mysql`；包安装时禁用 |
| 数据库用户名 / 密码 | 数据库连接凭据（密码加密存储）|
| 数据库端口 | 默认按厂商（MySQL 3306 等）|
| 数据库版本 | 自动检测 |

保存后自动检测连通性，在线设备自动采集静态信息。

### 3.3 列表操作

- **搜索**：按设备名称/IP 筛选（Ctrl+F 聚焦）
- **筛选**：按设备类型、在线状态筛选（支持 URL 参数，仪表盘跳转联动）
- **检测**：单台"检测"按钮；顶部"检测全部"
- **编辑 / 复制 / 删除**：行末按钮。**复制**保留类型/厂商/SSH/模板等配置，清空静态信息和 IP（新设备需重新检测），名称追加"(副本)"
- 右键菜单同样提供编辑/复制/检测/删除

### 3.4 自动检测机制

- **存活检测**：TCP 探测 SSH 端口。后台每 5 分钟全量轮询；离线设备每 90 秒快轮询尽快发现恢复
- **静态信息采集**：设备上线且缺静态信息时自动 SSH 采集。同一设备 10 分钟冷却期内不重复检测，防止状态抖动反复 SSH。非认证类失败（超时等）自动重试一次
- 网络设备仅支持 H3C、飞塔自动采集型号/SN；华为/思科/锐捷暂不支持自动采集，可手填

## 4. 巡检模板与命令库

切换到 **巡检模板** 页，包含三个标签页：巡检模板、命令库、报告模板。

### 4.1 命令库

内置网络设备（H3C/华为/思科/锐捷/飞塔）、Linux、数据库（MySQL/PostgreSQL/Oracle/SQL Server/达梦/Docker 容器）预置命令，按厂商分 tab、按类别分组折叠：

- 类别：版本信息、系统时钟、CPU、内存、硬件、存储、接口、VLAN、日志、协议、VPN、高可用、安全、无线、磁盘、网络、服务、进程、定时任务、通用

支持新增/编辑/删除命令，标记是否需要 root（sudo 提权）。

### 4.2 巡检模板

定义对设备执行哪些巡检命令。

1. 点击 **"+ 新建模板"**
2. 填写名称、选厂商、型号（可选）、描述、关联报告模板
3. 右侧"可选命令"按类别分组折叠，**点击命令**即添加到左侧已选列表
4. 左侧已选命令支持**拖拽排序**（顺序即报告中展示顺序）、删除
5. 保存

> 模板命令只保留命令本身，不再区分"巡检项/静态信息"——静态信息采集已独立于模板自动运行。模板支持**复制**，名称追加"(副本)"。

## 5. 报告模板

定义 DOCX 报告的版式。**按设备大类**（网络/Linux/数据库）而非厂商区分，因为同大类设备的报告结构一致。

### 5.1 编辑

1. 切换到 **报告模板** 标签页
2. 新建或编辑模板，选分类（网络/Linux/数据库）
3. 配置区块：封面、基本信息字段、巡检结果列定义、AI 分析、总体评估、页眉页脚
4. 基本信息/列定义支持字段勾选、拖拽排序
5. 右侧 A4 实时预览

### 5.2 字段

基本信息字段按分类不同：网络设备含 sysname/model/sn 等；Linux/数据库含 hostname/内核版本/CPU/内存等。

## 6. 执行巡检

批量创建巡检任务，连接设备执行命令并实时追踪。

### 6.1 创建批次

1. 切换到 **巡检执行** 页
2. 点击 **"+ 新建批次"**
3. 填写批次名称，勾选设备（多选），可选"创建后自动开始"
4. 创建并开始 / 仅创建

> 创建前会校验：设备是否在线、是否有 SSH 凭据、是否关联模板。离线/无凭据设备会提示，可移除后继续。

### 6.2 运行与监控

- 每台设备独立 SSH 会话，**多设备并发**
- 右侧面板：设备列表、各记录状态、进度
- 容器部署的数据库设备，巡检命令自动用 `docker exec <容器名>` 包装执行
- 操作：执行 / 暂停 / 停止 / 重启 / 重试（单条失败记录）

## 7. 报告管理

基于巡检结果生成 DOCX 报告。

### 7.1 流程

1. 选中左侧批次
2. 选择分析模式：
   - **AI 评判**：逐条命令输出送 AI，返回状态判定（正常/提示/注意/严重），自动生成报告
   - **人工评判**：跳过 AI 直接生成报告
3. 单设备：记录行"下载报告"
4. 批次：顶部"下载综合报告（ZIP）"或"下载综合报告（合并）"

### 7.2 报告内容

DOCX 含封面、基本信息表、巡检结果表（序号/项目/巡检内容/评判结论）、AI 分析总结、总体评估。多厂商批次按每设备匹配的报告模板生成。报告用 docx-rs 直接生成，不依赖 MS Office。

## 8. 工具箱

独立网络运维工具，不依赖已录入设备。

| 工具 | 功能 |
|------|------|
| 子网计算器 | CIDR 网段计算、IP 范围、掩码转换 |
| 存活扫描 | ICMP ping + TCP 135/445 回退，CIDR 网段扫描 |
| TCP 端口扫描 | 指定端口范围 TCP connect 扫描，常用服务预设标签（SMB/FTP/邮件/远程登录/数据库等）|
| UDP 端口扫描 | UDP 探针 + ICMP Port Unreachable 检测（DNS/SNMP/NTP/NetBIOS 等协议识别）|
| **路由跟踪** | 调用系统 traceroute/tracert，显示每跳节点 IP、延迟、归属地（离线 IP 库，一键下载）|
| WEB 检测 | HTTP/HTTPS 连通性和响应状态码 |
| SNMP v2c | Community + OID GET |
| SNMP v3 | USM 认证（MD5/SHA1/SHA256）+ 加密（DES/AES128）|
| Zabbix Agent | 被动模式探测（agent.ping/version/hostname）|

> 所有 IP 输入框强制英文输入法（imeMode disabled），避免中文输入法误输入。

### 8.1 端口扫描

输入目标 IP，选择端口（手动输入或点击预设标签）和超时，点击"开始扫描"。

**TCP 预设标签**：常用端口 / Web 端口 / 数据库端口 / SMB 文件共享 / FTP / 邮件服务 / 远程登录 / 基础设施 / 全端口范围

**UDP 预设标签**：常用 UDP / DNS+DHCP+NTP / SNMP 管理 / 发现服务 / SMB NetBIOS / Syslog TFTP

### 8.2 路由跟踪

输入目标 IP 或域名（支持域名自动解析），设最大跳数（默认 30）和每跳超时（默认 1000ms），点击"开始跟踪"。

结果表格显示：
- **跳数**：1, 2, 3...
- **节点 IP**：该跳路由器 IP，`*` 表示超时无响应
- **归属地**：国家 省市 运营商（需下载 IP 库）
- **延迟**：毫秒，`*` 表示超时

**IP 归属地**：
- 首次使用时提示"IP 归属地库未安装"，点击 **"一键下载"** 即可自动下载（~11MB）并启用，无需重启
- 也可手动下载 `ip2region_v4.xdb` 放到程序可执行文件同目录
- 路由跟踪本身不依赖 IP 库，未下载也能使用（归属地列显示为空）
- Linux 需先安装 `traceroute` 命令：`sudo apt install traceroute`

> **注意**：路由跟踪可能需要数十秒，视网络跳数而定。Windows 需 tracert（系统自带），Linux 需 traceroute。

## 10. 漏洞扫描

漏洞扫描模块用于检测目标主机上的已知安全漏洞。支持两种检测模式：

### 10.1 CVE 数据库管理

漏洞扫描依赖 CVE 数据库进行版本匹配。首次使用前建议下载离线 CVE 库：

1. 进入漏洞扫描页面
2. 在顶部状态栏点击 CVE 数据库旁的 **下载** 按钮
3. 等待下载完成（约 200MB，后台自动解压导入）
4. 下载后状态变为"已就绪"，支持离线查询

CVE 库默认使用在线 API 查询，下载本地库后优先使用本地数据。

### 10.2 扫描模式说明

| 模式 | 端口范围 | 说明 |
|------|---------|------|
| 快速扫描 | Top 88 常用端口 | 约 10 秒，覆盖 SSH/HTTP/MySQL/Redis 等常见服务 |
| 全端口扫描 | 1-65535 | 约 1-2 分钟，并发 TCP 连接 |
| 指定端口 | 用户自定义 | 输入 `22,80,443,8080-8090` 等格式 |

**扫描流程**：

```
端口扫描 → Banner 抓取 → 服务识别 → CVE 数据库匹配（显示关联漏洞列表）
```

结果展示：
- **总体评级**：Critical / High / Medium / OK
- **端口列表**：每个开放端口显示服务名称和版本
- **CVE 详情**：可逐条展开查看，点击 CVE ID 跳转 MITRE 详情页，标注 CVSS 评分、修复版本、EXP 公开状态

### 10.3 nuclei 验证引擎

版本匹配可能存在误报（已修复但版本号未变的情况）。如需精确验证漏洞是否真实存在，可安装 nuclei 验证引擎：

1. 在漏洞扫描页面顶部状态栏点击 **安装（~60MB）**
2. 自动从 CVE 服务下载 nuclei 二进制 + 漏洞模板
3. 安装完成后状态变为"已就绪"
4. 后续扫描将自动对每个开放端口发探测包验证

**工作原理**：

```
安装引擎后扫描流程：
端口扫描 → Banner 抓取 → CVE 库匹配 + nuclei 探针验证
                              ↓
                         仅返回确认存在的漏洞
```

> 注意：nuclei 验证耗时较长（每端口数秒至十几秒），但结果更准确。
> 未安装引擎时自动回退到 CPE 版本匹配模式。

## 11. 常见问题排查

### Q: Windows 安装后打开没反应 / 闪退

1. `Win+R` 输入 `%TEMP%\inspection-debug.log` 回车
2. 查看日志最后一行：

| 日志最后一行 | 问题 | 解决 |
|-------------|------|------|
| 无日志文件 | 程序无法启动 | 检查杀软是否拦截，重新安装 |
| `WebView2 未安装` | 缺少 Runtime | 安装包会自动引导，或手动下载[离线安装器](https://go.microsoft.com/fwlink/p/?LinkId=2124703) |
| 有弹窗报错 | panic | 截图发给开发者 |

### Q: SSH 连接失败 / 卡死

- 检查 IP、端口、用户名、密码
- 老旧设备 SSH 旧算法已内置兼容（libssh2 + OpenSSH 后备）
- TCP 端口不通时直接返回，不再反复试算法（避免长时间等待）
- 检查防火墙放行 SSH 端口

### Q: 数据库版本未自动获取

- **容器部署**：确认"容器名"填写正确（如 `mysql`），程序用 `docker exec <容器名>` 执行
- **包安装**：确认宿主机已安装对应数据库客户端（mysql/psql 等）
- 确认数据库用户名/密码正确
- 检测失败会在日志（`logs/inspection.log`）记录具体原因：密码错 / 容器未运行 / 客户端未安装

### Q: 设备静态信息未自动获取

- 确认设备在线（绿色状态）
- 确认 SSH 凭据已保存
- 网络设备仅 H3C/飞塔支持自动采集，华为/思科/锐捷需手填
- 点击设备行"检测"按钮手动触发

### Q: 同 IP 添加第二台设备提示冲突

- IP 唯一性按**设备类型**判断。同 IP 不同类型（如服务器 + 数据库）允许，同类型内不可重复

### Q: 报告生成失败

- 确认巡检记录已完成（status=completed）
- AI 评判需检查 API Key 有效、网络可达
- 报告不依赖 MS Office

### Q: 全新安装后数据库异常

- 关闭程序
- 删除数据库：Windows `%APPDATA%\inspection-rust\inspection.db` | Linux `~/.local/share/inspection-rust/inspection.db`
- 重启自动重建
