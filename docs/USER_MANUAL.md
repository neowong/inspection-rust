# 运维巡检系统用户操作手册

## 1. 系统用途

本系统用于通过 SSH 连接网络设备与 Linux 服务器，批量执行巡检命令，结合 AI 生成巡检分析，并输出可编辑的 DOCX 巡检报告。当前报告以 Word/DOCX 为最终交付格式，支持报告模板在线编辑和实时预览。

## 2. 基本流程

```text
配置 AI → 维护命令库 → 创建报告模板 → 创建巡检模板 → 添加设备 → 执行巡检 → AI 分析 → 生成 DOCX 报告
```

## 3. AI 配置

1. 打开“系统设置”。
2. 在 AI 配置区域新增模型配置。
3. 填写供应商、模型 ID、API Key、Base URL（如需）。
4. 点击“激活”。
5. 可使用健康检查确认模型可用。

> 没有激活的 AI 配置时，巡检仍可执行，但无法生成 AI 评判结果。

## 4. 命令库维护

进入“巡检模板 → 命令库”。

每条命令包含：

- 厂商：H3C / 华为 / 思科 / 锐捷等。
- 命令文本：例如 `display version`。
- 描述：会作为报告里的“项目”名称，例如“查看设备版本”。
- 分类：版本、CPU、内存、接口、日志等。

建议命令说明填写为面向报告阅读者的中文项目名，而不是命令本身。

## 5. 报告模板

进入“巡检模板 → 报告模板”。

### 5.1 新建报告模板

点击“新建模板”，配置：

- 模板名称
- 适用厂商（可选，空表示通用）
- 封面标题、副标题、主色调
- 设备信息字段
- 巡检明细表列
- 巡检总结
- 页眉、页脚

右侧为 A4 实时预览。

### 5.2 默认报告列

默认巡检明细表为 4 列：

| 列 | 说明 |
|---|---|
| 序号 | 自动生成 |
| 项目 | 来自命令库“描述” |
| 巡检内容 | 还原真实终端输出，格式为 `<sysname>command` + 设备输出 |
| 评判结论 | AI 状态、发现、建议整合显示 |

默认输出截断 15 行，可在模板中调整。

### 5.3 页眉页脚

- 页眉默认显示报告名称，并在下方显示单条横线。
- 页脚默认显示页码，并在上方显示单条横线。
- 页脚支持 `{{page}}` 和 `{{total}}`。

## 6. 巡检模板

进入“巡检模板 → 巡检模板”。

### 6.1 新建巡检模板

1. 选择厂商。
2. 添加命令。
3. 拖拽调整命令顺序。
4. 设置每条命令的用途。
5. 选择关联报告模板。

### 6.2 命令用途

每条命令可设置：

#### 巡检项

- 会执行。
- 默认显示到报告。
- 会参与 AI 分析。
- 适合 CPU、内存、接口、NTP、日志等正式巡检内容。

#### 静态信息

- 会执行。
- 默认不显示到报告。
- 不参与报告明细和 AI 分析。
- 用于提取设备静态字段。

支持提取字段：

| 字段 | 说明 | 示例命令 |
|---|---|---|
| `sysname` | 设备 CLI 提示符名称 | `display current-configuration \| include sysname` |
| `model` | 型号 | `display device manuinfo` |
| `serial_number` | SN | `display device manuinfo` |
| `manufacturing_date` | 出厂日期 | `display device manuinfo` |

例如 H3C 获取 sysname 的命令：

```text
display current-configuration | include sysname
```

输出：

```text
sysname aHope
```

报告中会使用：

```text
<aHope>display ntp status
...
```

但该静态命令本身不会显示在报告明细表中。

## 7. 设备管理

进入“设备管理”。

### 7.1 添加设备

填写：

- 设备名称
- IP 地址
- 厂商
- SSH 用户名 / 密码 / 端口
- 关联巡检模板

H3C 设备可自动检测：

- 型号
- SN
- 出厂日期
- Sysname（终端提示符）

这些信息会保存为设备资产信息。后续巡检时，如果模板中配置了静态信息命令，则本次巡检的静态信息快照会优先用于报告。

## 8. 执行巡检

进入“巡检执行”。

1. 选择设备或模板。
2. 创建巡检批次。
3. 点击运行。
4. 系统按设备并发执行命令。
5. 前端会定时刷新执行进度。

巡检结果中会保存：

- 报告展示命令输出：`command_outputs`
- 本次静态信息快照：`static_info`
- AI 评判结果：`command_judgments`

## 9. 报告管理

进入“报告管理”。

### 9.1 AI 分析

- 单条记录可点击“AI 分析”。
- 批次可点击“AI 分析全部”。

AI 分析会生成每条命令的状态、发现和建议。

### 9.2 生成报告

单条记录：

- 点击“生成报告”。
- 系统生成 DOCX。
- 点击“下载”保存到本地。

批次报告：

- “下载 ZIP”：每台设备一份 DOCX，打包为 ZIP。
- “下载合并 DOCX”：所有设备合并到一个 Word 文档，每台设备从新页开始。

### 9.3 生成目录（合并报告）

合并报告的“目录”页默认是空的，需要用 WPS/Word 手工生成一次带页码的目录。报告里的设备标题已用“标题1”、章节标题（设备信息/巡检明细/总结）已用“标题2”样式标记，可直接被目录收集。

**WPS 操作：**

1. 打开生成的合并报告 DOCX。
2. 把光标定位到“目录”页的空目录区域。
3. 菜单“引用 → 目录 → 自动目录”（或右键目录区域 → 更新域 → 更新整个目录）。
4. 目录会自动列出所有设备标题和章节标题，并带页码。

**Word 操作：**

1. 打开 DOCX，定位到目录页。
2. 菜单“引用 → 目录 → 自动目录 1”。
3. 若提示更新域，选“是”。

生成后如增删了设备或重新生成报告，需再次“更新域”刷新页码。

> 说明：由于生成 docx 的库限制，无法在生成阶段就把页码写死进目录，必须由 WPS/Word 在打开后计算分页并填充，因此需要这一步手工操作。

## 10. 报告输出规则

### 10.1 巡检内容

报告里的“巡检内容”列会还原终端输出：

```text
<sysname>display version
H3C Comware Software, Version ...
...
```

如果设备输出中第一行已经回显了命令本身，例如：

```text
display version
H3C Comware Software, Version ...
```

系统会自动去掉这行重复回显。

### 10.2 AI 评判

“评判结论”列会整合状态、发现、建议：

```text
【注意】
CPU 使用率偏高
建议：持续观察 CPU 负载趋势
```

## 11. 常见问题

### Q：为什么静态信息命令没有出现在报告里？

因为它的用途是提取报告需要的静态字段，不属于正式巡检项。请在巡检模板中将该命令设置为“静态信息”，并取消“显示到报告”。

### Q：报告里的 `<sysname>` 不对怎么办？

检查巡检模板是否配置了能提取 `sysname` 的静态信息命令，例如 H3C：

```text
display current-configuration | include sysname
```

并勾选提取字段 `sysname`。

### Q：报告输出太长怎么办？

在报告模板中调整”输出截断行数”，默认 15 行；设置为 0 表示不截断。

## 12. Linux 服务器巡检

本系统支持 Linux 服务器（Ubuntu/CentOS/Rocky/Debian 等）的 SSH exec 通道巡检：

1. 在”设备管理”添加设备，设备类型选择”服务器”，厂商选择对应 Linux 发行版。
2. 系统通过 exec channel 执行命令，支持 `sudo -S` 提权（需要设备 SSH 密码）。
3. 自动检测服务器信息（hostnamectl / os-release / nproc / dmidecode）。
4. 多连接并行（4 worker）执行命令，比单连接串行快 3-4 倍。
5. 支持提取 `cpu_cores` 和 `memory_gb` 静态信息字段。

> Linux 命令需预先在命令库中添加（厂商=Linux 等），并在巡检模板中勾选 `needs_root` 标记 sudo 命令。

## 13. 设备账号检测

v3.3.0 新增：每台设备支持 SSH 认证状态独立检测：

- 在线设备会显示双徽章：在线状态 + 账号状态
- 检测状态包含：账号正常 / 账号错误 / 无法连接 / 连接超时 / 解析失败 / 缺少凭据 / 账号未验证
- 设备列表支持按设备类型分类筛选（网络设备 / 安全设备 / 服务器）
- 批量检测所有设备时，在线设备会自动触发静态信息采集
