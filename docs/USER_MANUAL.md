# 运维巡检系统用户操作手册

## 1. 系统简介

本系统是面向运维工程师的桌面巡检工具，通过 SSH 连接网络设备（H3C/华为/思科/锐捷/飞塔）、Linux 服务器和数据库，批量执行巡检命令，调用 AI 分析结果并生成可编辑的 DOCX 巡检报告。

### 1.1 核心能力

| 能力 | 说明 |
|------|------|
| 设备管理 | 集中管理网络设备、安全设备、服务器、数据库资产 |
| 巡检模板 | 可视化模板编辑器，内置 85+ 预置命令，支持拖拽排序 |
| 批量巡检 | 多设备并发 SSH 执行，实时进度追踪，支持暂停/重试 |
| AI 分析 | 集成 OpenAI / Anthropic / DeepSeek，逐条命令评判 |
| 报告生成 | 可编辑 DOCX 报告，支持 AI 评判/人工评判，单设备/批量/合并 |
| 工具箱 | 存活扫描、TCP/UDP 端口、WEB 检测、SNMP v2c/v3、Zabbix Agent |
| 日志分析 | 设备日志解析与 AI 分析 |
| Linux 巡检 | exec channel 执行，sudo 提权，4 路并行连接 |

### 1.2 支持的设备类型

| 类型 | 说明 | 典型厂商 |
|------|------|----------|
| 网络设备 | 交换机、路由器 | H3C、华为、思科、锐捷 |
| 安全设备 | 防火墙、负载均衡 | 飞塔 (FortiGate) |
| 服务器 | Linux 服务器 | Ubuntu、CentOS、Rocky、Debian |
| 数据库 | 数据库实例 | MySQL、PostgreSQL、Oracle、Redis |

---

## 2. 基本流程

```text
配置 AI → 维护命令库 → 创建报告模板 → 创建巡检模板 → 添加设备 → 执行巡检 → AI 评判 → 导出 DOCX 报告
```

## 3. AI 配置

进入”系统设置”页面。

### 3.1 添加 AI 模型

1. 点击”新增配置”。
2. 填写：
   - **供应商**：OpenAI / Anthropic / DeepSeek 等。
   - **模型 ID**：如 `gpt-4o`、`claude-sonnet-4-20250514`、`deepseek-chat`。
   - **API Key**：对应供应商的密钥。
   - **Base URL**：留空使用官方地址；自定义填入完整 URL。
3. 点击”激活”。
4. 可使用”健康检查”确认模型可用。

> **注意**：没有激活的 AI 配置时，巡检仍可执行，但无法使用”AI评判”功能。可使用”人工评判”跳过 AI 分析直接生成报告。

### 3.2 支持的 AI 供应商

| 供应商 | Base URL | 备注 |
|--------|----------|------|
| OpenAI | `https://api.openai.com` | GPT-4o、GPT-4o-mini 等 |
| Anthropic | `https://api.anthropic.com` | Claude Sonnet、Haiku 等 |
| DeepSeek | `https://api.deepseek.com` | OpenAI 兼容格式 |

## 4. 命令库维护

进入”巡检模板 → 命令库”页面。

### 4.1 命令字段

每条命令包含：

| 字段 | 说明 | 示例 |
|------|------|------|
| 厂商 | 所属设备厂商 | H3C |
| 命令文本 | SSH 执行的命令 | `display version` |
| 描述 | 中文说明，会作为报告”项目”列 | “查看设备版本” |
| 分类 | 命令分类标签 | 版本、CPU、内存、接口 |
| 需要 root | 是否需要 sudo 提权（Linux 命令用） | 否 |

### 4.2 操作说明

- **添加**：选择厂商 → 填写命令和描述 → 保存。同厂商下不允许重复命令，不同厂商可以有相同命令。
- **删除**：删除后命令不会在下次启动时被种子数据恢复（墓碑机制）。
- **种子命令**：系统内置 85+ 条预置命令（H3C/华为/思科/锐捷/飞塔/Linux），首次启动自动填充。

### 4.3 填写建议

- 描述字段建议填写为面向报告阅读者的中文项目名，而不是命令本身。
- 例如：描述写”查看设备版本”，不要写”display version”。

## 5. 报告模板

进入”巡检模板 → 报告模板”页面。

### 5.1 新建报告模板

点击”新建模板”，配置以下内容：

| 配置项 | 说明 |
|--------|------|
| 模板名称 | 模板的标识名称 |
| 适用厂商 | 可选，留空表示通用模板 |
| 封面标题 | 报告封面大标题 |
| 封面副标题 | 报告封面小标题 |
| 主色调 | 封面和标题的颜色主题 |
| 设备信息字段 | 报告中显示哪些设备信息（名称、IP、型号、SN 等） |
| 巡检明细列 | 报告表格的列定义 |
| 巡检总结 | 是否显示总结区域 |
| 页眉 / 页脚 | 报告页眉页脚内容 |

右侧为 A4 实时预览，修改即时生效。

### 5.2 默认报告列

默认巡检明细表为 4 列：

| 列 | 说明 |
|---|---|
| 序号 | 自动生成 |
| 项目 | 来自命令库”描述”字段 |
| 巡检内容 | 还原真实终端输出，格式为 `<sysname>command` + 设备输出 |
| 评判结论 | AI 状态、发现、建议整合显示 |

默认输出截断 15 行，可在模板中调整；设置为 0 表示不截断。

### 5.3 页眉页脚

- 页眉默认显示报告名称，下方显示单条横线。
- 页脚默认显示页码，上方显示单条横线。
- 页脚支持变量：`{{page}}` 当前页、`{{total}}` 总页数。

### 5.4 多厂商模板

不同厂商的设备可使用不同的报告模板。系统在生成报告时会根据设备厂商自动匹配对应的模板。

## 6. 巡检模板

进入”巡检模板 → 巡检模板”页面。

### 6.1 新建巡检模板

1. 选择厂商。
2. 从命令库添加命令。
3. 拖拽调整命令顺序。
4. 设置每条命令的用途。
5. 选择关联报告模板。

### 6.2 命令用途

每条命令可设置两种用途：

#### 巡检项

- 会执行。
- 默认显示到报告明细。
- 会参与 AI 分析。
- 适合 CPU、内存、接口、NTP、日志等正式巡检内容。

#### 静态信息

- 会执行。
- 默认**不显示**到报告明细。
- 不参与报告明细和 AI 分析。
- 用于提取设备静态字段。

支持提取字段：

| 字段 | 说明 | 示例命令 |
|---|---|---|
| `sysname` | 设备 CLI 提示符名称 | `display current-configuration \| include sysname` |
| `model` | 型号 | `display device manuinfo` |
| `serial_number` | SN | `display device manuinfo` |
| `manufacturing_date` | 出厂日期 | `display device manuinfo` |
| `cpu_cores` | CPU 核心数（Linux） | `nproc` |
| `memory_gb` | 内存大小（Linux） | `sudo dmidecode -t memory` |

### 6.3 sysname 的作用

报告中的巡检内容会还原终端输出，格式为 `<sysname>command`：

```text
<aHope>display ntp status
Clock status: synchronized
...
```

如果没有配置 sysname 提取命令，报告中会省略设备提示符。

## 7. 设备管理

进入”设备管理”页面。

### 7.1 添加设备

填写以下信息：

| 字段 | 必填 | 说明 |
|------|------|------|
| 设备名称 | 是 | 设备的标识名称 |
| IP 地址 | 是 | SSH 连接地址 |
| 设备类型 | 是 | 交换机/路由器/防火墙/负载均衡/服务器/数据库 |
| 厂商 | 是 | 根据设备类型自动切换厂商列表 |
| SSH 用户名 | 是 | SSH 登录用户名 |
| SSH 密码 | 是 | SSH 登录密码（加密存储） |
| SSH 端口 | 否 | 默认 22 |
| 关联模板 | 是 | 绑定巡检模板 |

### 7.2 自动检测

设备保存后会自动触发：

1. **连通性检测**：TCP 连接设备 SSH 端口，判断在线/离线。
2. **静态信息采集**：在线设备自动 SSH 检测型号、SN、sysname 等。

支持自动检测的厂商：

| 厂商 | 检测命令 | 提取字段 |
|------|----------|----------|
| H3C | `display device manuinfo` | 型号、SN、出厂日期 |
| 飞塔 | `get system status` | 型号、SN、主机名 |
| Linux | `hostnamectl` / `lscpu` / `dmidecode` | 主机名、OS、CPU、内存 |

### 7.3 批量检测

点击”检测全部”按钮：

1. 并发 TCP 检测所有设备状态（5 秒超时）。
2. 在线设备自动触发静态信息采集（3 路并发，已有信息的设备跳过）。
3. 状态栏显示检测进度。

### 7.4 设备列表筛选

- **类型筛选**：网络设备 / 安全设备 / 服务器 / 数据库
- **状态筛选**：在线 / 离线 / 未知
- **搜索**：按设备名称或 IP 搜索

## 8. 执行巡检

进入”巡检执行”页面。

### 8.1 创建批次

1. 点击”+”按钮。
2. 填写批次名称（默认带日期时间）。
3. 选择要巡检的设备。
4. 可选择”自动开始”立即执行。

### 8.2 执行过程

- 每台设备独立 SSH 连接，多设备并发执行。
- 前端每 3 秒自动刷新进度。
- 巡检过程中可暂停/停止/重试。

### 8.3 执行结果

巡检完成后保存：

| 字段 | 说明 |
|------|------|
| `command_outputs` | 命令输出（JSON），报告展示用 |
| `static_info` | 本次巡检的静态信息快照 |
| `status` | 完成/失败/停止 |

### 8.4 重新巡检

- **重新巡检**：重置所有记录为 pending，重新执行。
- **重试单条**：仅重试失败的设备。

## 9. 报告管理

进入”报告管理”页面，左侧选择巡检批次。

### 9.1 批量评判

批次工具栏三个按钮：

| 按钮 | 功能 |
|------|------|
| **AI评判** | AI 分析所有记录 → 逐一生成每台设备的单份 DOCX |
| **人工评判** | 跳过 AI 分析，直接生成所有设备的单份 DOCX |
| **下载综合报告** | 合并所有单份报告为一个 DOCX 并弹出保存对话框 |

> “下载综合报告”在单份报告生成前为灰色（禁用），生成后变为可点击。

### 9.2 单设备下载

每台设备行末尾有”下载”按钮：

- 报告已生成 → 可点击下载单份 DOCX。
- 未生成 → 灰色禁用，需先点击批次”AI评判”或”人工评判”。

### 9.3 记录详情

点击设备行可展开详情，查看：

- 巡检命令输出（完整终端输出）
- AI 分析结果（逐条命令评判）
- 日志分析（可选）

### 9.4 生成目录（合并报告）

合并报告的”目录”页默认是空的，需要用 WPS/Word 手工生成一次带页码的目录。报告里的设备标题已用”标题1”、章节标题已用”标题2”样式标记，可直接被目录收集。

**WPS 操作：**

1. 打开生成的合并报告 DOCX。
2. 把光标定位到”目录”页的空目录区域。
3. 菜单”引用 → 目录 → 自动目录”。
4. 目录会自动列出所有设备标题和章节标题，并带页码。

**Word 操作：**

1. 打开 DOCX，定位到目录页。
2. 菜单”引用 → 目录 → 自动目录 1”。
3. 若提示更新域，选”是”。

> 说明：由于 docx 生成库的限制，无法在生成阶段就把页码写死进目录，必须由 WPS/Word 在打开后计算分页并填充。

## 12. Linux 服务器巡检

### 12.1 基本流程

1. 在”设备管理”添加设备，设备类型选择”服务器”，厂商选择对应 Linux 发行版。
2. 系统通过 exec channel 执行命令，支持 `sudo -S` 提权（需要设备 SSH 密码）。
3. 自动检测服务器信息（hostnamectl / os-release / nproc / dmidecode）。
4. 多连接并行（4 worker）执行命令，比单连接串行快 3-4 倍。

### 12.2 sudo 命令

Linux 命令需预先在命令库中添加（厂商=Linux 等），并在巡检模板中勾选”需要 root”标记 sudo 命令。

系统会自动通过 `sudo -S` + stdin 密码方式提权。

### 12.3 自动检测字段

| 字段 | 检测命令 | 说明 |
|------|----------|------|
| sysname | `hostnamectl` | Static hostname |
| model | `hostnamectl` / `cat /etc/os-release` | Operating System |
| cpu_cores | `nproc` / `lscpu` | CPU 核心数 |
| memory_gb | `sudo dmidecode -t memory` | 物理内存总量 |

## 13. 设备账号检测

每台设备支持 SSH 认证状态独立检测：

- 在线设备会显示双徽章：在线状态 + 账号状态。
- 检测状态包含：账号正常 / 账号错误 / 无法连接 / 连接超时 / 解析失败 / 缺少凭据 / 账号未验证。
- 设备保存时自动触发检测，结果写入设备的 `auth_status` 字段。

## 14. 常见问题

### Q：为什么静态信息命令没有出现在报告里？

因为它的用途是提取报告需要的静态字段，不属于正式巡检项。请在巡检模板中将该命令设置为”静态信息”，并取消”显示到报告”。

### Q：报告里的 `<sysname>` 不对怎么办？

检查巡检模板是否配置了能提取 `sysname` 的静态信息命令，例如 H3C：

```text
display current-configuration | include sysname
```

并勾选提取字段 `sysname`。

### Q：报告输出太长怎么办？

在报告模板中调整”输出截断行数”，默认 15 行；设置为 0 表示不截断。

### Q：删除的命令重启后又回来了？

v3.4.0 已修复此问题。删除命令时会写入墓碑记录，种子数据不会再恢复已删除的命令。

### Q：添加命令时提示”已存在”？

同厂商下不允许重复命令（如 H3C 下不能有两条 `display version`）。不同厂商可以有相同命令。

### Q：桌面图标显示为透明？

v3.4.0 已修复。图标已改为 RGB 白色背景，不再使用透明通道。
